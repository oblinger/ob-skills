# Track

Capability trait declaring that an anchor is actively driven through a planning + backlog lifecycle (the "drive loop").

## What it is

**Track declares that an anchor is actively driven through a planning + backlog lifecycle — the "drive loop."** Adding the Track trait turns on everything loosely associated with the anchor's planning tree: a backlog with workflow states, feature docs, optional roadmap / PRD / system-design / questions, and the drive-loop skills that operate over them (`/groom`, `/triage`, `/crank`, `/mint`, `/land`, `/fortify`, `/feature`, `/ask`, `/audit q`).

Track is **orthogonal to what the anchor *is***. `Code` says "this is a code project"; `Track` says "this project is run through a backlog and planning system." A code project, a writing project, a skill, or a knowledge hub can all be Tracked — or not. The two compose: `traits: [Code, Track]` is the common case; `traits: [Track]` alone is a non-code project that still has a backlog (e.g. a wiring / ops effort).

## How it's detected

- **Trait:** `Track` appears in the anchor's `.anchor` `traits:` list. One-line lookup; no file inference (per [[CAB Aspects]]).
- Corroborating (not the detection rule): a Track anchor has a `{NAME} Track/` folder containing at minimum `{NAME} Backlog.md`. Detection is still the declared `traits:` field — the folder is the manifestation, not the test.

## Wiring an anchor for Track

1. **Declare the trait.** Add `Track` to the `.anchor` `traits:` list, e.g. `traits: [Code, Track]` or `traits: [Track]`.
2. **Create the tree root.** `{NAME} Docs/{NAME} Track/` with the dispatch page `{NAME} Plan.md` per [[CAB Plan Dispatch]] (H1 `# {NAME} Plan`, dispatch table, one row per planning doc that exists).
3. **Create the required core.** `{NAME} Backlog.md` with the horizon H2s (`## Now` / `## Next` / `## Later`, plus `## Active` / `## Ready` as used) per [[CAB Backlog]] + [[backlog]]. An empty-but-well-formed backlog is valid.
4. **Add optional parts as needed** — Features/, Roadmap, PRD, System Design, etc. — each per its facet spec. Don't pre-create empties; add a part when its first real content arrives.
5. **Link from the anchor page.** Add a `Plan` row to the anchor page's dispatch table pointing at `{NAME} Plan.md` (per [[CAB Anchor Page]]).
6. **The drive loop now applies.** `/groom`, `/triage`, `/crank`, `/mint`, `/land`, `/feature`, `/ask`, `/audit q` all operate on this anchor; its section appears in `~/ob/kmr/Q.md`.

Minimum viable Track anchor = trait declared + `{NAME} Track/{NAME} Plan.md` + `{NAME} Backlog.md`. Everything else is additive.

## Format

Structural expectations on a Track anchor as a whole:

- The tree lives under `{NAME} Docs/{NAME} Track/` (one level below the standard Docs folder).
- Exactly one Backlog per anchor (the Backlog facet is cardinality-one).
- Source order of backlog rows is preserved; `/triage` and `Q.md` never reorder the backlog file (re-ordering is a Q.md-only behavior).
- Feature docs are `F<NNN> — <Title>.md` (zero-padded three digits) under `{NAME} Features/`, monotonic-forever numbering per [[CAB Backlog]].

## Constraints

- **Co-requirement.** The `Track` trait **requires the `Backlog` facet** — declaring `Track` mandates a `{NAME} Backlog.md`. (A Track anchor with no backlog is a violation; audits flag it.)
- **Cardinality: many / composable.** `Track` is a capability layered on top of the anchor's identity trait — it is *not* one of the single-valued `{Simple, Topic, Code, Paper}` set.
- **Composition.** Legal with `{Topic, Code, Paper, Skill}`. **Excludes `Simple`** (a Simple anchor is "just a folder + marker, nothing else" — adding a drive loop makes it not Simple). See the composability matrix in [[CAB Aspects]].
- **Exactly one Backlog** (inherited from the Backlog facet's own cardinality constraint).

## Expected Usage

- **Most common pairing: `Code + Track`** — a code project run through a backlog. Also common: `Skill + Track` (e.g. [[SKA]] itself), and `Track` alone for a non-code ops/wiring effort.
- **`Track` is the prerequisite for the drive-loop skills.** If an anchor isn't getting `/crank` / `/groom` attention, it's usually because it lacks `Track` (or its backlog is empty).
- **Typical scale:** 5–50 active backlog rows; more than that usually wants `## Later` / `## Icebox` to absorb the tail.
- **Don't pre-create empty optional parts** — a Roadmap/PRD/Icebox should appear when there's real content, not as scaffolding.
- **Cadence is separate.** *Whether* an anchor is Tracked (this trait) is independent of *how aggressively* it's driven (the Drive vs Lean cadence axis, [[F077]]). Don't conflate `Track` (has a drive loop) with `Drive` (the aggressive cadence mode).
