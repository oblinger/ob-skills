# BRIEF

- **What this file is** — the auto-generated dispatch catalog of every skill folder under `~/.claude/skills/`. One row per discovered skill (or skill-adjacent file), pointing at the skill's `SKILL.md` (or markdown variant). The H1 row carries the breadcrumb; rows below are the catalog.
- **Don't hand-edit rows.** This file is maintained by the slug-scan / anchor-scan tooling (`cab-slug-scan`, `ha --dump`). Rows are discovered, not curated. Manual edits get clobbered on the next scan. To change a row, fix the underlying skill folder (rename, add a description, etc.) and rescan.
- **Row format** — `[[skill-name]] | <description>` (wiki-link form) or `[skill-name](hook://...)` (hook form for non-vault targets). Description is one short phrase pulled from the skill's frontmatter; blank is allowed when the skill hasn't supplied one. Mixed link forms reflect how each row was discovered — do not normalize by hand.
- **Order is discovery order, not alphabetical.** New skills append at the bottom on first scan. Don't re-sort manually; if ordering matters for a downstream surface, build a curated dispatch elsewhere (e.g. an `SKL <Topic> Guide`) and link from there.
- **The `Facets` / `CAB-parts` / `compile` rows are aggregator rows** — their second cell is a comma-separated wiki-link list, not a description. Preserve their structure when scans regenerate; they're populated from separate inventories (CAB facets, compiled-skill artifacts).
- **Don't add non-skill content here.** Project docs, CAB specs, and guides belong in their own catalogs ([[CAB Base]], [[SKL]], etc.). If something shows up as a row that isn't a skill, the fix is in the scanner's discovery rules, not in this file.
- **When NOT to expect a row** — internal helpers without a `SKILL.md` (e.g. shared scripts), retired skills (move to `SKILL-retired`), and compiled artifacts that are derived from a source skill rather than first-class.
