# Harbor: A Self-Hosted Media Server — 2026-06-01

## Abstract

Harbor is a self-hosted media server that ingests, transcodes, and streams a personal media library from commodity hardware. This paper presents its three-component architecture and the single-catalog design choice that keeps a one-operator deployment legible.

## Architecture

Harbor splits into three components: **Ingest** (watches a drop folder, fingerprints and imports files), **Serve** (a streaming API with on-the-fly transcode), and **Operate** (backups, retention, health). Each is independently restartable; state lives in a single SQLite catalog.

## Results

On commodity hardware Harbor imports a 1,400-file library in under four minutes and sustains three simultaneous transcoded streams. The single-catalog design keeps the operator's mental model small.
