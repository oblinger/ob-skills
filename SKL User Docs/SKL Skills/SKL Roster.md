# /roster

State-of-the-work summary for the current anchor. Walks the anchor's `{NAME} Backlog.md` and prints In Progress / Ready / Backlog buckets (with counts and item list) plus an Icebox count. The Icebox is counted but never listed.

**When to use:** "roster", "show roster", "what's on the backlog", or any moment you want to see the current state of the work without context-switching into the file.

See `~/.claude/skills/roster/SKILL.md` for the full runbook. Item rendering follows the B-number convention from [[CAB Backlog]].
