# /viz pdf — Markdown to PDF via pandoc + tectonic

Convert a markdown file directly to PDF. Use when polish is not required — a quick share, an automated pipeline, or default typography is fine.

For controlled output where you want to tweak fonts/layout/page breaks before delivery, use `/viz docx` instead (md → docx → polish in Word/Pages → export PDF).

## Tools

- `pandoc` — `brew install pandoc`. Verify with `pandoc --version`.
- `tectonic` — modern Rust-based LaTeX engine, lighter than MacTeX (~200MB vs 4GB). `brew install tectonic`. Verify with `tectonic --version`.

Alternative pandoc PDF engines: `xelatex`, `lualatex`, `wkhtmltopdf`. Default to `tectonic` — best output quality with smallest install.

## Usage

```
/viz pdf <input.md> [--output <path>]
```

- **input.md** — source markdown file (required).
- **--output** — output path. Default: `~/ob/data/MyDesk/<slug>-<YYYY-MM-DD-HHMM>.pdf`.

## Command

```bash
pandoc <input.md> -o <output.pdf> --pdf-engine=tectonic
```

If the input contains wiki-links, preprocess first (same sed pattern as `/viz docx`).

## Styling via YAML metadata

For consistent styling across docs without a template file, include a pandoc YAML metadata block at the top of your markdown:

```yaml
---
geometry: margin=1in
fontsize: 11pt
mainfont: "Charter"
sansfont: "Inter"
monofont: "JetBrains Mono"
colorlinks: true
linkcolor: "blue"
---
```

These fields drive LaTeX/tectonic rendering. For deeper control (custom headers, footers, multi-column, page numbering schemes), fall back to `/viz docx` → polish in Word → export, or write a custom pandoc LaTeX template.

## Wiki-link handling

Same as `/viz docx`:

```bash
sed -E 's/\[\[([^|]+)\|([^]]+)\]\]/\2/g; s/\[\[([^]]+)\]\]/\1/g' input.md > /tmp/input-clean.md
pandoc /tmp/input-clean.md -o output.pdf --pdf-engine=tectonic
```

## Workflow

1. **Compute output path** — slug + timestamp under MyDesk (unless `--output` specified).
2. **Preprocess wiki-links** if any `[[X]]` appears in the input.
3. **Run pandoc** with `--pdf-engine=tectonic`.
4. **Print output path** so the user can `open <output.pdf>`.

## When to use

- Quick share — no need to tweak.
- Automated pipeline — md generated by another tool, rendered to PDF as the final step.
- You've set YAML metadata defaults and they work for everything.

For per-document polish, use `/viz docx` instead.

## Verification

```bash
printf '# Smoke test\n\nBody text. **Bold** and *italic*. A [[wiki-link]] here.\n' > /tmp/smoke.md
sed -E 's/\[\[([^|]+)\|([^]]+)\]\]/\2/g; s/\[\[([^]]+)\]\]/\1/g' /tmp/smoke.md > /tmp/smoke-clean.md
pandoc /tmp/smoke-clean.md -o /tmp/smoke.pdf --pdf-engine=tectonic
open /tmp/smoke.pdf
```

Expected: clean single-page PDF opens in Preview; wiki-link renders as plain "wiki-link."
