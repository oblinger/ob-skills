When the search agent runs against a **book** — fiction, non-fiction, technical, reference — these are the rules that apply.

The most common verb here is **describe** — producing a book dossier (summary, core arguments, chapter overview, key takeaways). Legacy `/research book` is fully subsumed; the output format (publisher / page count / awards / chapter list / takeaways / author background) is pre-baked.

`survey` (compare books in a space — "best epistemics books", "top distributed-systems textbooks") and `find` (pin a specific edition / ISBN) fall back to the verb defaults; pair-specific knowledge can land in `SRC rules/survey-book.md` / `SRC rules/find-book.md` as patterns emerge.
