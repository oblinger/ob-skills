- **F001 — Migrate legacy anchor-root pages to CAB RID Page format** [ ] — Many existing anchor-root pages (`{NAME}.md` files across the kmr tree) still use the pre-CAB format: `:>> [[kmr]] → …` breadcrumb line at top, `n::` or `desc::` inline description, no YAML frontmatter, no dispatch table. The new format per `[[CAB RID Page]]` uses YAML frontmatter with `description:`, a dispatch table as the first element (with `-[[NAME]]-` slug and `->` breadcrumb placeholder the rebuild expands), and `^^^` / `...` separators for auto-populated rows.

  Scope: kmr-wide. Expected to touch thousands of files — every anchor root, and knock-on effects for any page that references them. Known legacy examples include [[SYS]], `Meta.md`, [[Log]], [[Receipts]], and much of the old Topic/LST/MY trees.

  Prerequisite before execution: work through the format rules for the distinct anchor cases together (RID vs. no-RID, simple vs. full, old-style `hook://p/Name` leaf-links, pages with non-standard body structures, etc.). Need a clear migration rubric per case before any bulk pass. The cost of getting the rubric wrong and propagating it across 1000s of files is high.

  No urgency — active work can continue in new anchors using the new template without blocking on this migration.

# BRIEF

- **This is CAB's own backlog** — work items scoped to maintaining and evolving the CAB specification (facets, rules, parts, disciplines). Not a vault-wide backlog and not a substitute for per-anchor backlogs.
- **Inclusion test** — a row belongs here only if the work changes CAB itself: a facet spec, a rule, a part, the base structure, or a CAB-level discipline. Cross-anchor migrations driven by a CAB change (e.g. F001) are in-scope as tracking items.
- **Structure follows [[CAB Backlog]] facet spec** — horizon H2s (`## Now` / `## Next` / `## Later`) plus `## Ready` for items pulled forward; `## Done` accumulates completed work. Workflow brackets (`[Ready]`, `[ ]`, etc.) live inline on each row per [[workflow]].
- **Naming conventions** — features are `F<NNN>` (zero-padded triple-digit), bugs `B<n>`, conditions/findings `C<NN>`. Each top-level item is a single bullet with an `^B-…` block-id where it needs to be link-targeted (e.g. `^B-QFix`).
- **Audit-fix sub-bullets nest under their parent finding** — QFix-style residuals from `/audit q --fix` hang off `B-QFix` as `**C<NN>**` sub-bullets with the full source path + line + finding text. Don't promote these to top-level rows.
- **Don't pile non-CAB work here** — anchor-specific work belongs in that anchor's backlog; project-wide policy belongs in `CLAUDE.md`; markdown-rendering rules belong in [[R-markdown]]. Keep this file lean and CAB-scoped.
