# CAB Repository Structure

Convention spec for the standard files, folders, and `justfile` recipes a CAB code repository carries.

See also [[CAB Code Repository]] for the structural description.

## Key Repository Files
- **README.MD** — Brief project description, installation, quick start
- **CLAUDE.MD** — Claude Code project instructions; coding conventions, key patterns
- **PYPROJECT.TOML** — Dependencies, build config, project metadata (Python)
- **JUSTFILE** — Task runner; common tasks like `just build`, `just test`, `just docs`
- **MKDOCS.YML** — MkDocs documentation site configuration (if using MkDocs)

## site/ Folder
Generated by documentation build. Should be gitignored. Deployed to GitHub Pages.

## Justfile Standards

The `justfile` is the standard way to manage project tasks. Use [just](https://github.com/casey/just) as the task runner.

### Standard Recipes
- **SETUP** — Fix up dev environment so all symlinks and dependencies are correct
- **BUILD** — Perform an incremental build
- **REBUILD** — Clean build from scratch
- **TEST** — Run the basic test suite
- **TESTALL** — Run all maintained tests
- **LINT** — Run linters (ruff, mypy, etc.)
- **CHECK** — Run all checks (lint + test)
- **DOCS** — Build documentation
- **INSTALL** — Install dependencies
- **DEV** — Install in development mode
- **CLEAN** — Remove build artifacts

### App-Specific Recipes
Add as needed for applications:
- **RUN** — Run the application
- **RERUN** — Stop the running app, rebuild, then launch
- **BUILDRUN** — Build then run (without stopping first)

# BRIEF

- **This file is a CAB convention spec** — it prescribes the standard repository skeleton (key files, the `site/` folder, justfile recipes) that any CAB-conformant code repository should carry. Treat edits here as normative for downstream anchors.
- **NOT for per-project deviations** — project-specific recipes, file layouts, or build steps belong in that project's own `CLAUDE.md` / `justfile` / repo README, not here. Only promote a pattern to this page once it is genuinely cross-project.
- **Inclusion test for a new entry** — the file, folder, or recipe must be expected to appear in *most* CAB code repositories (or be a named optional with clear semantics). One-off tooling does not belong.
- **Naming conventions are load-bearing** — file names (`README.MD`, `JUSTFILE`, `PYPROJECT.TOML`, `MKDOCS.YML`) and recipe names (`SETUP`, `BUILD`, `TEST`, `LINT`, `CHECK`, `DOCS`, …) are quoted verbatim by tooling and by sibling specs; do not rename without updating cross-references (notably [[CAB Code Repository]]).
- **Keep the structural description in [[CAB Code Repository]]** — this page is the catalog of standard files and recipes; deeper structural rules (folder layout, layering, gitignore policy) live there. Don't duplicate that material here.
- **Two-section shape — Files and Recipes** — keep new content under one of the existing H2s (`## Key Repository Files`, `## Justfile Standards`) and preserve the Standard vs. App-Specific split for recipes so readers can scan the required core separately from optional add-ons.
