# RULESET R-doc
include:: [[R-markdown]], [[R-file-association]], [[R-ruleset]], [[R-brief]], [[R-discussion]], [[R-log]], [[R-messages]], [[R-query]], [[R-doc-structure]], [[R-prd]], [[R-architecture]], [[R-testing]], [[R-decisions]], [[R-stories]]
description:: Everything checked when auditing a document — markdown + file conventions plus the doc-facet rulesets (Doc Structure, Ruleset, Brief, Discussion, Log, Messages, Query) and the design-doc facet rulesets (PRD, Architecture, Testing, Decisions, Stories).

The umbrella that **`/audit doc <path>`** resolves ([[F161 — Rule-driven audit engine — resolve, run, judge|F161]]). The convention sets (`R-markdown`, `R-file-association`) apply to every document (`always`); the facet sets fire only on their `where::` targets — `R-doc-structure` on every authored doc, a `# RULESET` block → `R-ruleset`, a `# BRIEF` → `R-brief`, a Discussion heading → `R-discussion`, a `* Log.md` → `R-log`, a `* Messages.md` → `R-messages`, a `* PRD.md` → `R-prd`, a `* Architecture.md` → `R-architecture`, a `* Testing.md` → `R-testing`, a `* Decisions.md` → `R-decisions`, a `* Stories.md` / `US-*.md` → `R-stories`. The per-rule `where::` selector does the binding, so an included ruleset only fires on its own file kind. Add a doc facet's ruleset to the `include::` line to bring it into `/audit doc`.
