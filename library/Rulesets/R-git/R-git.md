# RULESET R-commit-discipline
include::
where:: anchor
description:: commit cadence and message hygiene — when to commit, never amend, no boilerplate, co-author trailer

Commit-discipline rules extracted from the global `/dev` instructions (2026, F133 sweep). They govern *when* an agent commits and what a commit message looks like — distinct from the PR / branch-protection rulesets the umbrella will also carry. An anchor's Git **aspect** (`Commit` / `PR` / `NoGit`) selects the cadence; these rules apply wherever commits happen. Format of this set: [[FCT Ruleset]].

### RULE R-commit-discipline-01 — Commit at logical boundaries without asking (stated)

In an anchor running **Commit** mode, the agent commits at logical work boundaries on its own — it never asks the user "want me to commit?" (the question is itself a violation). PR mode commits freely on the branch; NoGit has nothing to commit.

**Why:** asking permission to commit is friction with no benefit; the anchor's Git aspect already answers the question (see [[FCT Query]] R-query-14).

### RULE R-commit-discipline-02 — Never amend a commit (stated)

A landed commit is never amended; corrections land as new commits on top.

**Why:** amending rewrites shared history and loses the audit trail of how the work progressed.

### RULE R-commit-discipline-03 — No "generated by Claude" boilerplate in commit messages (checked)
check:: regex_absent (?i)generated (with|by) claude

Commit messages carry no "generated by / with Claude" or similar AI-attribution boilerplate; messages stay brief and describe the change.

**Check pattern:** the commit message body contains no `generated (with|by) claude`-style boilerplate line.

### RULE R-commit-discipline-04 — Branch before committing on a protected default (stated)

When work begins on a protected default branch (e.g. `main`), the agent branches first rather than committing directly to it.

**Why:** keeps the default branch clean and reviewable; protects against accidental direct pushes.

### RULE R-commit-discipline-05 — Don't commit or push unless asked (when the anchor isn't in an autonomous Git mode) (stated)

Outside an anchor with an explicit autonomous Git aspect (Commit / PR), the agent commits or pushes only when the user asks. The aspect is what licenses unprompted commits.

**Why:** commit autonomy is opt-in per anchor; a generic project does not grant it by default.
